    exploring_mars v0.1.0 ExploringMars.Mission.Direction

    This module defines functions that create and operate on directions in the probe's coordinate space.

    This module should change if the coordinate representation used in the problem changes in degrees of freedom - for instance, if we decide the probe could move in 3D space rather than 2D, or if the probe could move in 8 directions instead of 4. In that case, we would need to also change the Coordinate module in order to handle the new directions.

    This coupling between the Direction and Coordinate modules is, for now, acceptable, because the Directions to be handled are few. If we were to choose a case where we would have more possible Directions, it might be worthwhile to create an Axis module which would specify the axes of movement available to the probe, and make both Direction and Coordinate modules depend on the Axis module - the Direction module would produce which axis or axes correspond to each direction, and the Coordinate module would know how to update the coordinate according to motion on each axis.

    That way, we would mitigate the amount of code that should be changed in the case of a change in coordinate system degrees of freedom.
